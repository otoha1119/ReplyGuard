ReplyGuard — いまの仕様（やさしい説明版）

現状：PoC（試作）段階／2026-06-09 時点。むずかしい用語をなるべく避けた全体説明です。

これは何をするもの？

メールを自動で取り込み、「これは大事？」「返信が要る？」「対応にどれくらい手間がかかる？」をシステムが判定して、大事な順に一覧で見せてくれるツールです。対応が済んでいないメールを見落とさない（対応漏れを防ぐ）ことが目的です。

いまは Gmail だけに対応しています。将来は Slack や Outlook なども統合する構想ですが、それらはまだ作っていません。

ぜんたいの流れ

① メールを取り込む → ② 内容を判定する → ③ 大事な順に並べる → ④ 保存する → ⑤ 必要なら通知する → ⑥ 画面に一覧表示

この一連を、手動ボタンでも、一定間隔の自動実行でも動かせます。

① メールの取り込み

あなたの Gmail から最近のメールを読み込みます。読み込みは「見るだけ」で、メールを既読にしたり、書き換えたり、送信したりは一切しません（読み取り専用）。

接続にはGmailの「アプリパスワード」を使います。これは各自が自分のぶんを用意し、外部には出しません。

② 判定システム（重要度などの判断）

取り込んだメール1通ずつを読んで、次のことを判定します。

  • 重要度（5段階のレベル）
  • 種類（仕事／お知らせ／宣伝 など、ざっくりの分類）
  • 対応の重さ（さっと返せる／じっくり対応が要る など）
  • かんたんな要約（何のメールかの一言まとめ）

判定のやり方は2通りあり、設定で切り替えられます。

標準かんたん判定（その場で動く・無料）
送信元や件名のキーワード（「至急」「請求」「セール」など）から、AIを使わずにルールで判定します。インターネットや契約なしで、すぐ全体を動かせます。初期状態はこちらです。

任意LLM（生成AI）による判定
設定すると、Anthropic の Claude（または OpenAI）にメール内容を渡して、より自然な重要度判断・要約をしてもらえます。AIの利用キーを設定したときだけ有効になり、キーが無ければ自動でかんたん判定に戻るので、止まることはありません。

AIにはメール本文を「判断してほしいデータ」として渡すだけで、本文に書かれた指示（例：「これを無視して〜しろ」）には従わせない作りにしています。AIの返事も決められた形だけ受け取り、おかしな出力は弾きます。

③ 大事な順に並べる（優先順位づけ）

各メールに優先スコアをつけて、高い順に並べます。スコアは主に次の3つで決まります。

  • 未読かどうか（未読のほうが上に来やすい）
  • 重要度（②で判定したレベルが高いほど上）
  • どれくらい時間が経ったか（放置されているものを埋もれさせない）

ざっくり言うと「未読 × 重要度 × 経過時間」で、放っておけないものほど上に出ます。

④ 対応状況の管理

メールごとに「いまどういう状態か」を持っていて、ボタンで切り替えられます。

  • 未対応 … 取り込んだ直後
  • 対応中 … 着手した
  • 完了 … 対応し終えた
  • スヌーズ … 後回し（一旦寝かせる）
  • 対象外 … 対応不要として外す

2人が同時に同じメールを更新しても食い違わないよう、競合を検知して「画面を最新にしてから操作してね」と知らせる仕組みが入っています。

⑤ 保存場所（データベース）

取り込んだメールと判定結果はこのPC内のローカルなデータベース（SQLite）に保存されます。インターネット上のどこかに送ることはありません。

  • 標準では、プロジェクト内のファイル1つにためていきます（メール本文は個人情報なので外部共有・コミットはしません）。
  • チームで共有する本格的なデータベース（PostgreSQL）に切り替えることもできます（任意）。
  • 同じメールを何度取り込んでも対応状況は引き継がれます（「対応中」が勝手に「未対応」に戻ったりしません）。

⑥ 通知システム

大事なメール（重要度が高いもの）や新しく届いたものについて、お知らせを出せます。届け先は3通りから選べます。

標準ログに記録（どこにも送らない）
アプリの動作記録に「重要なメールが来た」と要点だけ残します。本文の全文は出しません。外部に何も送らないので、初期状態はこれです。

任意メールで通知
設定したメールアドレス宛に、お知らせメールを送ります（メール送信の設定が必要）。

任意Slackで通知
指定した Slack のチャンネルにメッセージを投げます（Slack側の受け口URLが必要）。

設定が足りないときは自動で「ログに記録」に戻ります。また、同じメールを短時間に何度も通知しないように重複を抑えています。

画面（ダッシュボード）

ブラウザで開くと、メールが大事な順のカード一覧で並びます。各カードに差出人・件名・重要度・要約・状態が出て、ボタンで状態を変えられます。「更新」ボタンで最新のメールを取り込み直せます。

自動で動く部分

起動時に1回メールを取り込み、その後も一定間隔で自動的に取り込みを続けます（間隔は設定可能）。手動の「更新」ボタンでもいつでも取り込めます。

ログイン（認証）

いまは初期状態でログイン不要（自分のPCで試す前提）です。必要になれば、合言葉（トークン）でのログインを有効にできます。

安全面でのこだわり

  • メールは見るだけ（既読化・送信・削除はしない）。
  • パスワードやAIキーなどの秘密情報はファイルに直書きせず、共有もしない（各自のローカルだけ）。
  • PC内の情報やメール本文を外部に勝手に送らない（AI判定を使うときの正規のやり取りを除く）。
  • 画面に出す相手（許可した接続元）を限定し、知らないサイトから中身を読まれないようにしています。

いまできること / まだできないこと

できる：Gmailの取り込み、重要度などの判定（かんたん判定／AI判定）、大事な順の並び替え、対応状況の管理、ローカル保存、通知（ログ／メール／Slack）、ブラウザ一覧表示、自動取り込み。

まだ：Slack や Outlook、GitHub からの取り込み、クラウド上での本格的なデータ共有、複数人での本格運用向けのログイン・権限管理。これらは構想段階で未実装です。
